# Contributing

Thanks for your interest in contributing.

## Development setup
1. `composer install`
2. `composer test`
3. `composer analyse`
4. `composer fix:dry`

## Guidelines
- Keep changes focused and small.
- Follow existing naming and structure.
- Add tests for new behavior.
- Run the test suite before submitting a PR.

## Commit style
- Use clear, present-tense messages (e.g., "Add Khalti refund support").
